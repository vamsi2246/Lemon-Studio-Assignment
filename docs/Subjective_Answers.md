# Subjective Answers

## Q1. Difference between Keyword Search and RAG

### Answer:

Keyword search finds results using exact words entered by the user.

RAG (Retrieval-Augmented Generation) first retrieves relevant information from documents and then uses an AI model to generate an answer based on that information.

I would use RAG when working with PDFs, company documents, knowledge bases, or any application where users need accurate and contextual answers.

## Q2. What factors should be considered before deploying an LLM to production?

### Answer:

Before deployment, I would consider:

* Accuracy of responses
* Hallucination prevention
* Security and privacy
* Response speed
* API cost
* Scalability
* Monitoring and logging

These factors help make the system reliable and safe for users.

## Q3. Describe a technical project you worked on

### Answer:

I built an Enterprise Multi-Document RAG Assistant using React, FastAPI, FAISS, LangChain, and Gemini API.

The application allows users to upload multiple PDF documents and ask questions based on their content.

One challenge was retrieving relevant information when multiple documents were uploaded. I solved this by using vector embeddings and document-based filtering.

Another challenge was integrating AI APIs and deployment. I resolved it through debugging, proper error handling, and testing.

This project helped me learn full-stack development, RAG architecture, vector databases, AI integration, and deployment.
